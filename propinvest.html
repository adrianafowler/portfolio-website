PropInvest

Reponsive website for property buyers. UI project

Overview:

Objective:

PropInvest is a UI study project of responsive web app that provides property buyers with information on properties of interest.

Context:

Real estate investment is an increasingly popular way for individuals to achieve financial security. It is an exciting and emotional experience, but often complicated. They are plenty of websites which provides buyers with information about properties, but are they making the whole process easy for buyers? Do they provide users with data which would help them to make right decision?

Requirements:

Sign in, create user profile, and input property criteria.
Search and filter available properties
Access comprehensive information about a given property (e.g., specs, value, previous sales, location, visuals, etc.) and its neighborhood.
Bookmark a property listing, property recommendations feature
Ability to contact real estate professional when wanting to move forward with a property (i.e., guided viewing, ready to invest).

The task:

Having been provided with UX research data, my task was to create a User Interface based on gathered information.
My first step was to apply UX methodology which would help me to create functional and useful prototype:

UX Design process:

Problem statement

Buyers new to the property market may struggle to get started without professional guidance which could result in time spent viewing properties which are out of their price range.

Solution:

Design a web app which will provide new property investment buyers with the expertise needed to get started efficiently.

Challenge:

Combine two major functions in one website: online property agency and investment advice.

Persona:

Rashida is a fictional persona, created from the research results , for whom I was designing PropInvest . Rashida would like to invest in property and secure financial future for herself and her family.

Features and services offered by PropInvest:

Search filters, watchlist, detailed information on properties, book viewing with calendar, contact agent with live chat, investment estimation calculators, create profile, subscribe for preferences.

User flows and sketches:

Flows helped me to predict how the user would interact with the product and how screens and features have to be designed in order for users to achieve their goals. I created 6 flows and drew sketches for each scenario.

Sketches - first low fidelity wireframes are a very convenient way to integrate through the early stage of the design. This helped me to focus on the main functions and features.

UI design step 1:

After completing the UX task, I had a good foundation to start the UI design process.
I began with creating a grid, which helped me transfer hand drawn sketches into digital wireframes.

Setting grid for different breakpoints and applying reponsive patterns:

Following material design as my guide I set 4 columns for mobile, 8 for tablet and 12 for desktop with column width for each resolution 60px. Taking ‘mobile first’ approach, I created a website for mobile resolution as a starting point and then created patterns and layout for larger devices like tablet and desktop.To define the smallest breakpoints I used Bootstrap framework which I find to be a great indicator for responsive design. Mobile first approach also helped me to shape Information Architecture of the website as designing for small resolution first, helps to prioritise Information.

Responsive property description page:

On the right hand side is an example of how I applied responsiveness to the property description page. Apart from creating responsive layout I also created responsive patterns. Responsive patterns determine how the design shifts from breakpoint to breakpoint, always presenting users with the most convenient functionality and layout based on the device they’re using to view the site or app. Property description page pattern: Top navbar (global menu): Quick access menu on mobile and tablet hides under hamburger menu. As desktop provides more space I’ve left menu elements out.

Layout:

Mobile design - Most elements are placed in one column.
Tablet allowed me to fit search results into two columns whereas desktop left space for a more larger photograph and for more elements being displayed.

UI Design step 2 - Creating visual identity

Mood board:

Mood board was my inspiration for colour schemes, photographs, illustrations, art work, fonts, icons, illustrations and UI elements.
It helped me to establish a style and visual direction. Creating a mood board was the first step in developing a visual identity for the project.

Characteristics of PropInvest mood board:

  • Dominant colour: dark blue
  • Style: elegant mixed - formal and informal
  • Imagery subjects: properties, money, graphs
  • Language: informal, informative, accessible/understandable for everyone.

UI Design step 3 - Text Style

After exploring several combinations for fonts I chose two typefaces: Libre Baskerville for headings and titles mainly and Source Sans Pro for body and elements.
Libre Baskerville which is serif typeface gave an informative and elegant feel to the website, Source Sans Pro completed the design with its neat sans serif look.

UI Design step 4 - colours, logo and loading screen:

Choosing colours for the design is a thoughtful task.
As colours convey certain information and trigger certain emotions it is a good idea to conduct a thorough research before making a decision.
I was experimenting with a variety of ‘safe’ and unusual combinations. After making a final decision, I’ve applied chosen colours onto different variations of logo design.
Then I used the logo as an element of the loading screen to keep visual identity.

Rationale behind choices:

Blue is elegant, reflects reliability and trustworthiness and is used very often by property sellers’ websites.
By adding orange, I intended to bring playfulness and divertion from "too formal" look.
Lime is a colour closely associated with confidence, high energy and is thought to promote feelings of liveliness, freshness, and creativity.
I found this colour as a perfect addition to create a contrast.

UI Design step 5 - Imagery:

Choosing the right images comes with a lot of consideration about messaging and the story you’d like to tell with a design.
It is important to remember that images convey emotion directly to the user.
I applied several different images and illustrations to the design, but the decision was not easy.
For me this was the most challenging task, but thank to UX Design tools such as preference testing I managed to overcome this.
I conducted a test and the final decision was made by gathering information from potential users.
Most of the participants chose photographs with warm colours.
Participants associated them with bright future and happiness.

UI Design step 6 - Icons:

Designing icons could be challanging too.
I followed the advice on creating a grid, which helped me to keep the proportions, then I exported icons as vectors and applied them to the design.
Icons were designed in Illustrator.

UI Design step 6 UI Elements:

Access Design System

High fidelity wireframes

After completing UX design process and creating UI visual identity, it was time to bring everything together and create high fidelity wireframes.
As the biggest obstacles were behind me, I still had to be aware of the purpose of the product and make sure it solves a problem and meets the design criteria which were:

  • Ability to save or mark properties I am interested in, so that I can easily revisit them.
  • Ability to create a profile containing all my property criteria, so that I only receive results most relevant to me.
  • Ability to contact the right people if I am interested in viewing a property, so that I can schedule a viewing to help find what I’m looking for, quickly.
  • Access to as much written and visual information as possible about properties I’m interested in, so that I can make an informed decision.
  • Ability to search and filter properties, so that I can find good matches based on my requirements.

Presenting final mockups, prototype, interactions and animations.

Finally, I could present my design as mockups.
For this I used Photoshop. I tried different mockup styles and I’ve chosen the one which I believe presents PropInvest project in the best way.
After that I made animations with Flino and Principle and then I created interactive prototype with InVision (access link below mockups)

Retrospective

Did I solve the problem?:

Problem: Buyers new to the property market may struggle to get started without professional guidance which could result in time spent viewing properties which are out of their price range. My design offers potential buyers with a home page and search feature with filters (results will show only relevant properties). This could save users’ time on sorting through properties which are irrelevant to them. I created chat, contact agent and book viewing features, so the user can communicate and receive professional advice. User can calculate potential investment profit using the calculator feature and gather information about properties with property description screen. Having all these features provides the buyer with professional guidance when making an investment.

What went well?

Creating a right mood board gave me a very good direction for UI design. This definitely made the UI process much easier.
Skills: Research, creating visual identity.
Process: UI Design
Solution: Give a lot of attention when creating a moodpboard. Having a relevant mood board makes UI design process easier.

User flows helped me to create wireframes quickly.

Skills: strategic thinking, user-centred design, research and analysis.
Process: UX design, understanding the user
Solution: Dividing flows into small chunks and creating sketches.

Calculators under tabs on property investment model screen are a great solution for people who plan to invest in property.

Skills: strategic thinking, user-centred design, ui design solutions.
Process: UX design, wireframing and prototyping.

What didn't go well?

Inconsistency in global navbar placement between mobile, tablet and desktop design. I had to change the design and placement of global menu in mobile version, so it keeps visual consistency throughout all resolutions, despite their responsiveness.
Skills: Sketch, visual design, ux design (functionality).
Processes: Designing high fidelity wireframes.
Skills gap: “Need more practice with responsive design.”
Solution: Redesign navbar and place it on the top, so it fits tablet and desktop. Always have other breakpoints in mind while designing for mobile.

What can be improved?

Skills: Icon design
Processes: Polishing design, creating icon set.
Skills gap: More experience in working with Illustrator could save time with future projects.
Solution: Illustrator tutorials, icon design tutorials.

What methods I used?

UX design process included:
Competitive research and analysis, stating a problem, creating persona, user flows, creating sketches which provided me a structure for UI design. The main challenge was time management. It took me three months to complete a project. I’ve learned that UI design requires great attention to detail, time and a computer with large RAM and fast processor (at least five programs were opened during my work).

Conquering the challenge:

Challenge: Combine two major functions in one website: online property agency and investment advice.
Solution:
On property description page under ‘invest’ tab, user is provided with more detailed information about property’s buying history. User can calculate estimated profit from buy to rent or buy to sell, using ‘calculators’ on the website. Other features of PropInvest which could help users to make a property investment decision are: ‘find the best areas to invest in your country’ page and ‘property prices history and prognosis for areas of interest’ page.
